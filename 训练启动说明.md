# 单卡 / 六卡训练启动说明（SemiDino）

本文给出“当前仓库这版代码”在**单卡 4090**和**六卡 3090**上的建议配置与启动方式，尽量减少反复改配置导致的问题。

## 1. 你需要重点关注的设置

### 1.1 `configs/SemiDino.py`

1) `BATCH_SIZE`  
- 含义：**单卡 batch size**（每个 GPU 的 batch）。  
- 你目前单卡 4090 能稳定跑，通常可以保持不变（当前是 `4`）。

2) `TARGET_NUM_GPUS`  
- 含义：你计划使用的 GPU 数量（当前已设为 `6`）。  
- 作用：参与 `auto_scale_lr.base_batch_size` 计算。

3) `auto_scale_lr`  
- 当前为：`dict(enable=True, base_batch_size=BATCH_SIZE * TARGET_NUM_GPUS)`。  
- 含义：以“6 卡 * 单卡 batch”为 LR 基准。  
- 效果：在 6 卡上保持当前 LR 基准不变；若换成其他卡数，LR 会按总 batch 线性缩放。

> 如果你只打算长期固定在 6 卡上训练，`BATCH_SIZE` 与 `TARGET_NUM_GPUS` 这两个值通常就够用了，其他训练主流程仍走官方 DINO 基线。

### 1.2 `SemiTrain.py`

1) `--launcher`  
- 单卡：`none`（默认）  
- 多卡（torchrun）：`pytorch`

2) `--work-dir`（可选）  
- 用于覆盖配置里的输出目录。

3) `--config`（可选）  
- 切换配置文件时使用，默认是 `configs/SemiDino.py`。

---

## 2. 单卡 4090 启动方式

### 推荐命令

```bash
python SemiTrain.py --launcher none
```

### 你通常不需要改的内容

- 不改 `TARGET_NUM_GPUS` 也能跑（只是 auto scale 会按当前卡数计算）。  
- 若你希望“单卡时 LR 也固定不变（不随卡数缩放）”，可把 `auto_scale_lr.enable` 改回 `False`。

---

## 3. 六卡 3090 启动方式

### 推荐命令（单机 6 卡）

```bash
torchrun --nproc_per_node=6 SemiTrain.py --launcher pytorch
```

### 可选命令（指定输出目录）

```bash
torchrun --nproc_per_node=6 SemiTrain.py --launcher pytorch --work-dir ./work_dirs/semi_dino_6x3090
```

### 六卡场景建议

1) 先保持 `BATCH_SIZE=4` 做首轮验证。  
2) 若显存紧张，再把 `BATCH_SIZE` 调小（如 3 或 2）。  
3) 调整 `BATCH_SIZE` 后无需手工改 LR：当前 `auto_scale_lr` 会自动按总 batch 线性缩放。

---

## 4. 关于 mmdet 导入顺序是否会引入新问题

当前脚本采用的是更稳妥的三段式：

1) 先检测是否已安装 `mmdet`；若有，则优先使用环境版本。  
2) 若没有，再尝试注入仓库 `thirdparty/mmdetection-3.3.0`。  
3) 若仍不可用，立即抛出清晰错误并终止，而不是在后续 Runner 构建阶段才出现隐晦错误。

此外，`register_all_modules` 仍在 `main()` 内执行，且在路径检查之后、`Runner.from_cfg` 之前执行，这能保证 registry 初始化时机正确。

---

## 5. 快速自检

在真正开训前建议至少执行：

```bash
python SemiTrain.py --help
python -m py_compile SemiTrain.py configs/SemiDino.py
```

若你想看脚本最终采用的是哪种 mmdet 来源（环境安装或 thirdparty），启动时日志会打印：

```text
[SemiTrain] launcher=..., mmdet_source=site-packages|thirdparty, work_dir=...
```
